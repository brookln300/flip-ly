### 2. Code Changes

**`app/lib/scrapers/types.ts`**
- Added `search_path?: string` to `CraigslistConfig`

**`app/lib/scrapers/craigslist.ts`**
- Added `CL_CATEGORIES` map (13 categories with labels + default event types)
- `scrapeCraigslist()` now reads `config.search_path` (defaults to `gms` for backward compat)
- `isItemListing` flag controls: event_date handling (null for items), URL path, logging
- Posting URLs now use correct category path (`/ela/d/...` instead of `/gms/d/...`)

**`app/lib/ai/enrich-listing.ts`**
- System prompt expanded with dual scoring rubric: sale events vs individual items
- Item scoring: 1-3 (at/above market), 4-5 (near market), 6-7 (20-40% below), 8-9 (40-60%+ below), 10 (collector pricing)
- `deal_score_reason` now includes resale value estimates (e.g. "DeWalt drill at $60, resells $120-150")
- `resale_flag` threshold loosened to 2x for priced individual items (was 3x)

**No changes needed to:**
- `app/api/cron/scrape/route.ts` — already dispatches by `source_type`, reads config, passes to scraper
- `app/api/listings/route.ts` — search already handles `event_type=listing`, null event dates, price filters
- `app/lib/redis.ts` — Price Bands, Resale Radar, Search Replay all work on any listing
- `vercel.json` — existing cron schedule covers these (scrape every 4h, enrich every 30m)
