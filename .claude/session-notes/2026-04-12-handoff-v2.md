## What We Did

### 1. Craigslist URL Fix (Critical Bug)
**Root cause:** SAPI `searchPath` codes (e.g., `cba` for collectibles) were used directly as CL URL path segments, but CL's URL routing uses different category codes (e.g., `clt`). 11 of 13 categories had mismatched codes causing 404s on every item listing link.

**Fix (3 parts):**
- `SAPI_TO_URL_CATEGORY` mapping — all 13 SAPI→URL codes verified by testing real posting IDs against CL routing
- URL construction now extracts area code from `decode.locations` via latLng prefix, slug from item array (handles different positions in gms vs item listings)
- DB migration: 3,065 existing broken URLs fixed via SQL replacement

**File:** `app/lib/scrapers/craigslist.ts` (lines 23-41 mapping, lines 174-202 URL construction)

**Verified:** Constructed URLs return 200 (correct area) or 301→200 (wrong area, auto-redirects). Zero 404s.
